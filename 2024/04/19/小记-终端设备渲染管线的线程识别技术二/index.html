ShuffleDog: Characterizing and Adapting User-Perceived Latency of Android Apps

an API-instrument-based approach to identify proper threads accurately at a low cost.

  • how a user interaction is processed in Android OS
    a touch event, in the form of a hardware event
    -> subsequently read by a kernel driver by means of interrupts
    -> format and send to the user space
    -> Input Reader : pre-processing
    -> Input Dispatcher determines the destination app to which the event is sent in the form of an OS touch event through IPC
    -> processes and …

  • a delay-critical thread：i) it processes user input directly or indirectly, and ii) it changes screen output directly or indirectly.
    meets the preceding two conditions at the same time

  • method
    API - inject codes; send info to Thread Tracker
    sharing mem - n annotate each lock, atomic object, and message with a unique ID; tracing 2 threads acquiring the same lock/atomic object/message

  • Limitation
    slow ->
    < context; decision > pairs in SQLite table;
    make a quick decision when the same execution context happens again;
    update / add new pairs; specify the max size of in-memory data size (default is 2 MB) and replacement policy of in-memory items (MRU or LRU13);
    records only incremental information(增量信息)

  • details
    Kernel-Space Modification;
    scheduling优化
