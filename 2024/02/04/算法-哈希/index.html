算法总结三——hash table
2024-02-04|算法

算法总结之三。参考代码随想录：https://www.programmercarl.com/

实现方式

通常会使用以下三种数据结构来表示哈希表：

数组

例：Leetcode242：给定两个字符串 s 和 t ，编写一个函数来判断 t 是否是 s 的字母异位词。

用一个Letter[26]表示每个字母出现的次数。

集合

Leetcode349

给定两个数组，编写一个函数来计算它们的交集。

用一个set集合（unordered_set:读写效率最高，并不需要对数据进行排序,数据不重复）表示。

Leetcode202

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」定义为：对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和，然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。如果 可以变为 1，那么这个数就是快乐数。

也使用unordered_set，重点在于循环如何终止：出现了相同的sum（个位数平方和）。

说明：适用于hash数组难以表示的情况，但是速度要比数组慢，set把数值映射到key上需做hash计算。

map

Leetcode1

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那两个整数，并返回他们的数组下标。

由于需要记住数组下表，所以需要带有key和value的map映射。

其他

三数之和和四数之和用双指针处理更好。分别为两层循环和三层循环。
