安卓的渲染

Graphics-aware Power Governing for Mobile Devices：2.1 Android Graphics Pipeline

process:

  1. hardware composer send VSync (hardware composer -> UI Thread)
  2. update begin; 60times/min
  3. UI Thread traverses the display list and creates drawing commands ( view hierarchy - render node - the display list for the corresponding view )
  4. drawing commands from the display list are stored in the deferred display list, RenderThread traverses the deferred display list and creates an optimized drawing command (-> RenderThread)
  5. RenderThread converts it into OpenGL commands
  6. delivers the command to the GPU through the GPU Channel (-> GPU)
  7. RenderThread sets a flag in the BufferQueue; released when the GPU rendering is complete (-> BufferQueue)
  8. SurfaceFlinger retrieves frames from the BufferQueue (->SurfaceFlinger)
  9. passes them to the hardware composer (-> hardware composer)
  10. passes the frame to the display device (done)

Study and Refactoring of Android Asynchronous Programming

Android provides three major async constructs: AsyncTask, IntentService and AsyncTaskLoader.

  • AsyncTask
    AsyncTask　<Params, Progress, Result>
    https://developer.android.google.cn/reference/android/os/AsyncTask?hl=en

    Params: 输入参数类型
    Progress: 执行的进度返回给UI线程的参数的类型
    Result: 异步任务执行完后返回给UI线程的结果的类型

  • IntentService
    https://developer.android.google.cn/reference/android/app/IntentService?hl=en
    继承于Service类；
    onHandlerIntent();
    创建HandlerThread();
    可以执行处理多个 Intent 请求，顺序执行多任务;
    任务结束后 IntentService 会自行停止，无需手动调用 stopService;

  • AsyncTaskLoader
    provides an AsyncTask to do the work.

An Analysis of Thread Scheduling in Mobile Device using Thread Interaction Graph

Panappticon : Event-based tracing to measure mobile application and platform performance

goals: tracing user-perceived transaction formed by the operations between the user input and display update

  • separating each execution trace into “atomic” intervals,
  • identifying the causal relationships between intervals,
  • identifying the initial (e.g., user input) and terminal (e.g., display update) intervals of user-perceivable transactions.

(a)intervals (b)mark events (c)constuct Relationship graph (d)full relationship graph

Architecture: a userspace logger, a kernelspace logger, an event collector, a server-side collector, and a user transaction analyzer.

relationship: causal relationship & temporal ordering.

  • Causal relationship: 实边、因果节点对
  • Temporal ordering： 虚边、when to end
    • Task-based threads: Looper thread wait for messages; one message belong to one execution interval
    • worker threads: one-time; automatically

Resource accounting:

  • add context switch, network and disk IO access events, synchronization events
  • analyzed the edges on the critical path
